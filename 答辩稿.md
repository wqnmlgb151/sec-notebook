# sec-notebook 答辩材料

> Web 安全渗透测试学习平台 | Vue 3 + Vite + GitHub API

---

## 一、整体介绍

**sec-notebook** 是一个面向 Web 安全初学者的在线学习平台，系统化讲解常见 Web 漏洞的原理与防御方案。采用 **HTML5 + CSS3 + JavaScript + Vue 3 + Vite** 技术栈，纯前端架构，部署于 Cloudflare Pages。

一句话：**用 Vue 3 单文件组件构建的、数据从 GitHub 动态抓取的 Web 安全知识库。**

## 四、路由设计

本项目采用**多页面导航**结构（非 Vue Router SPA），通过 `<a>` 标签实现页面间跳转。

### 导航结构

```
首页 (index.html)
  ├── 学习笔记 (list.html)          ← Vue SFC 渲染的独立页面
  │     └── 漏洞详情 (detail.html)   ← 笔记中相关内容链接到详情页
  ├── 漏洞详情 (detail.html)         ← 直接访问，支持 ?id= 参数定位
  ├── 安全实验室 (login.html)
  └── 关于项目 (about.html)
```

### 关键设计点

| 机制 | 实现 |
|------|------|
| **公共导航** | `components.js` 动态注入 Header + Footer 到所有页面，修改一处全局生效 |
| **当前页高亮** | `common.js` 的 `highlightCurrentNav()` 根据 `location.pathname` 匹配激活 |
| **Hash 定位** | list.html 用 `window.location.hash` 持久化笔记选择，支持刷新恢复 |
| **参数定位** | detail.html 读取 `?id=sqli` 自动滚动到对应漏洞章节 |
| **面包屑** | 每页内联面包屑导航，指示当前位置 |

### 为什么没有用 Vue Router？

项目 5 个页面中 4 个是纯静态内容页面（首页/详情/实验室/关于），不涉及组件切换。只有 list.html 需要 Vue 的数据驱动能力。如果引入 Vue Router 会把简单问题复杂化——需要把所有 5 个页面的 HTML 内容塞进一个 SPA，反而增加维护复杂度。当前方案每个页面独立加载，职责清晰，更适合这个规模的项目。

## 六、Vue 3 核心 API：ref 与 reactive

### 项目中的使用情况

本项目主要使用 **`ref()`**，未使用 `reactive()`。这是有原因的。

### ref() — 基本类型的响应式包装

```js
// 项目实例（App.vue）
const files = ref(filesArr)       // 数组 → ref 包装
const sel = ref(null)             // 对象 → ref 包装
const isLoading = ref(false)      // 布尔 → ref 包装
const showSidebar = ref(true)     // 布尔 → ref 包装
```

**特性**：
- 将任意值包装为响应式对象，通过 `.value` 读写
- 模板中自动解包，不需要 `.value`
- 可以整体替换值：`files.value = newArray`——这对于从 API 获取新数据后整体替换列表非常关键

### reactive() 为什么没用

```js
// reactive 写法（本项目未采用）
const state = reactive({
  files: filesArr,
  sel: null,
  isLoading: false
})
```

`reactive()` 适合固定结构的对象，但不能整体替换（`state = newState` 会丢失响应性），且解构时会丢失响应性。本项目需要在 API 返回后整体替换 `files` 数组，所以用 `ref()` 更合适。

### computed() — 派生状态

```js
// 项目实例（App.vue）
const currentIndex = computed(() => {
  if (!sel.value) return -1
  return files.value.findIndex(f => f.rawName === sel.value.rawName)
})
const hasPrev = computed(() => currentIndex.value > 0)
const hasNext = computed(() => currentIndex.value < files.value.length - 1)
```

**价值**：不用手动维护 `currentIndex`，`sel` 或 `files` 变化时自动重新计算。`hasPrev` 和 `hasNext` 进一步派生，模板中直接用 `:disabled="!hasPrev"`，逻辑清晰。

### watch() — 响应式回调

```js
// 项目实例（App.vue）— 替代旧版 setInterval 轮询
const stopWatch = watch(files, (val) => {
  if (val.length > 0) {
    stopWatch()
    const found = val.find(f => f.rawName === targetName)
    if (found) selectFile(found)
  }
})
```

**价值**：页面加载时 URL 带了 hash（如 `#01 代理配置.md`），但 files 可能还在从 API 获取。`watch` 监听 files 第一次填充后自动定位目标笔记。比旧版的 `setInterval` 轮询（每 100ms 查一次）更高效、更声明式。

### 数据驱动对比传统 DOM 操作

```
传统做法（jQuery 风格）：               Vue 3 做法（本项目）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━    ━━━━━━━━━━━━━━━━━━━━━━━━━
用户点击笔记                          用户点击笔记
  → 手动 fetch 数据                     → selectFile(f)
  → 手动 createElement 建 DOM            → sel.value = f
  → 手动 appendChild 插入                → 模板自动重新渲染
  → 手动改 class/href                   → v-if/v-for 自动更新
```

### 现存问题

1. **没有后端**：登录注册仅前端演示，无真实鉴权与数据持久化
2. **CSS 单文件过大**：1681 行 common.css 未按组件拆分
3. **无自动化测试**：所有功能靠浏览器手动验证
4. **highlight.js CSS 仍走 CDN**：atom-one-dark 主题未纳入构建
5. **学习笔记页与其他 4 页技术栈不一致**：list.html 走 Vite 构建，其他 4 页仍为纯静态 HTML

### 改进方向

1. 引入 Node.js/Python 后端实现真实用户系统
2. 将 CSS 按页面/组件拆分为多个文件，通过 Vite 的 CSS `@import` 聚合
