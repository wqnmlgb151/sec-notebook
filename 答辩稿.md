# 答辩演讲脚本 — sec-notebook

> 按顺序照着念就行，每页预估时间已标好。总时长约 5-8 分钟。

## 第 1 页：项目是什么（~40秒）

**打开首页 index.html，展示页面**

**你要说的：**

大家好，我的大作业题目是 **sec-notebook**，一个 Web 安全学习平台。

一句话概括：**它是一个纯前端的静态网站，用暗黑风格展示了 Web 安全漏洞的原理和防御方法。**

关键数字：5 个页面、19 篇学习笔记、6 大漏洞深度分析、手机/平板/电脑三端都能看。

---

## 第 2 页：用了哪些技术（~40秒）

**不用翻页，继续口头说**

**你要说的：**

技术栈非常简单：HTML5 + CSS3 + JavaScript + Vue 3。

- **HTML5** — 所有页面统一 Header/Nav/Main/Footer 结构
- **CSS3** — 70 多个自定义属性，做了一套完整的设计系统
- **JavaScript** — 导航交互、表单验证、Toast 通知
- **Vue 3** — 学习笔记的加载、搜索、Markdown 渲染
- **marked.js + highlight.js** — 把 GitHub 上的 .md 文件渲染成漂亮的网页
- **DOMPurify** — 防止 XSS 攻击

**全部依赖通过 CDN 引入，零构建工具，双击就能打开。**

## 第 3 页：5 个页面分别干什么（~2分钟）

**逐个打开页面展示**

### 3.1 首页
**打开 index.html，滚一下**

**你要说的：**

首页是第一印象。Hero 区域用了 Glitch 故障动画和霓虹发光效果，中间是 Bento Grid 便当盒布局展示了 5 个核心安全模块，下面是 19 篇笔记的入口卡片，底部有常用渗透工具的展示。

### 3.2 学习笔记页
**打开 list.html，点几篇笔记演示**

**你要说的：**

这是整个项目技术含量最高的页面。它是用 Vue 3 驱动的，会自动从我的 GitHub 仓库抓取 Markdown 笔记列表，点左边目录，右边实时渲染。支持 GFM 完整语法——表格、代码高亮、任务列表都行。

**演示操作：**
1. 点左边一篇笔记 → 右边出内容
2. 按 Ctrl+→ 翻到下一篇
3. 按 ESC 返回目录

**你要补充的：**

如果 GitHub API 挂了或者限流了，代码会自动回退到内置的 19 篇笔记列表，不会白屏。加载中显示骨架屏动画，加载失败有重试按钮——四种状态全都有处理。

### 3.3 漏洞详情页
**打开 detail.html，滚到 SQL注入**

**你要说的：**

详情页深入讲解了 6 个 OWASP 漏洞：SQL 注入、XSS、CSRF、SSRF、文件上传、IDOR。每个漏洞都按照"原理 → 攻击示例 → 防御方案"的结构来写。攻击示例用终端风格代码块展示，旁边标了严重度和 CWE 编号。URL 后面加 `?id=sqli` 可以直接跳到对应漏洞。

### 3.4 安全实验室
**打开 login.html，切一下登录/注册，输一下密码**

**你要说的：**

安全实验室演示了前端表单验证的完整实现。登录和注册两个面板互相切换，注册表单有邮箱验证、手机号验证、密码强度实时检测。

**演示操作：**
1. 在密码框随便敲几个字 → 强度条会变
2. 输两次不同密码 → 提示不一致

**你要补充的：**

页面上明确写了"仅作前端演示，不会向服务器发送凭据"——这是安全意识的一部分。

### 3.5 关于项目
**打开 about.html 快速扫一眼**

**你要说的：**

最后一页交代了项目背景、技术栈、设计理念和开发时间线。用了标签云展示设计关键词。

## 第 4 页：设计亮点（~1分钟）

**不用翻页，口头说**

**你要说的：**

说一下设计方面的考虑：

我选了**暗黑赛博玻璃态**这个视觉方向，而不是常见的浅色主题。原因是：
1. 和安全主题气质契合——黑客终端的感觉
2. 避免模板化——不是那种千篇一律的渐变 Hero + 卡片网格

具体实现了几个特效：
- **毛玻璃效果** — 卡片用 `backdrop-filter: blur()` 半透明
- **霓虹发光** — 重点元素用青色和紫色多层 `box-shadow` 发光
- **扫描线** — 整个页面有一层若隐若现的扫描线叠加
- **Glitch 故障动画** — 标题 hover 的时候文字错位 + 色彩分离

配色上用了深底色 `#020617`（几乎是纯黑），青色 `#06b6d4` 作为主强调色，红/橙/绿分别标识严重/高危/安全状态。

响应式做了 3 个断点：手机（≤768px）单列 + 汉堡菜单，平板（769-1024px）两列，桌面（≥1025px）完整布局。

## 第 5 页：技术难点（~1分钟）

**你要说的：**

讲两个我觉得比较有意思的技术点：

**第一个，动态内容 + 静态部署。** 这个项目是纯静态的，没有后端服务器。但笔记内容是动态的——通过 GitHub REST API 实时抓取我的学习笔记仓库，marked.js 在客户端渲染成 HTML。仓库里加了新笔记，网站自动就有，不用重新部署。API 挂了有硬编码回退，保证离线也能用。

**第二个，XSS 防护。** 因为用 `v-html` 渲染 Markdown 内容，有 XSS 风险。我做了三层防护：
1. DOMPurify 净化 marked.js 的输出，剥离危险标签
2. 每个页面配了 CSP 策略，限制脚本来源
3. 数据源只从指定 GitHub 仓库读取

## 备查：可能被问到的问题

**Q: 为什么选安全主题？**
A: 两个原因。一是 Web 安全是前端开发者必须了解的领域，以攻促防。二是黑暗风格和安全主题天然契合，设计上有发挥空间。

**Q: 为什么不直接用 WordPress/Hugo/Hexo？**
A: 课程要求手写前端代码，不能套模板。我选择了用 Vue 3 CDN 的方式，既满足了课程要求，又用上了现代框架。

**Q: 笔记内容是原创的吗？**
A: 笔记是我学习 Web 渗透测试时整理的，存放于 GitHub 仓库，项目通过 API 动态抓取。

**Q: 最大的收获是什么？**
A: 从零到一独立完成了完整的前端项目，掌握了 CSS 设计系统、Vue 3 数据驱动、安全编码（CSP/DOMPurify/XSS防护）这些实际工作中会用到的技能。
