## 一、项目概述

### 1.1 选题背景

网络安全威胁日益严峻，OWASP Top 10 漏洞仍然是 Web 应用面临的最大风险来源。本项目的初衷是**以攻促防**——将 Web 安全知识与前端开发技术结合，构建一个系统化、可视化的安全学习平台，帮助开发者理解常见漏洞的原理与防御方法。

### 1.2 项目定位

**sec-notebook** 是一个**暗黑赛博玻璃态 (Dark Cyber Glassmorphism)** 风格的 Web 安全学习平台。它不是一个后端服务，而是一个**纯静态前端项目**——无需构建工具，无需服务器，可直接部署到 Cloudflare Pages。

### 1.3 核心数据

| 指标 | 数值 |
|------|------|
| 页面数 | **5 个**（首页 / 笔记库 / 漏洞详情 / 安全实验室 / 关于） |
| 页面跳转 | **三级结构**（首页 → 笔记库 → 详情页） |
| CSS 设计令牌 | **70+** 个自定义属性 |
| 笔记数量 | **19 篇** Markdown 学习笔记 |
| 漏洞分析 | **6 大** OWASP 漏洞深度讲解 |
| 响应式断点 | **3 端**适配（手机 ≤768 / 平板 769–1024 / 桌面 ≥1025） |
| Git 提交 | **14** 次迭代提交 |

## 三、页面功能详解

### 3.1 首页 (`index.html`)

**设计目标**：第一眼建立「专业安全平台」的品牌印象

| 区域 | 内容 | 技术亮点 |
|------|------|----------|
| **Hero 首屏** | 霓虹辉光徽章 + 渐变标题 + Glitch 特效文字 | `@keyframes glitch` 文字错位 + 色彩分离动画 |
| **背景装饰** | 青/紫光球 + 扫描线 + SVG 噪点纹理 | `body::after` 固定叠加 + `feTurbulence` 滤镜 |
| **数据统计** | 19 篇笔记 / OWASP 标准 / 严重度分级 / GitHub 开源 | CSS Grid 四等分，hover 辉光上浮 |
| **Bento Grid** | 注入攻击 / XSS / 访问控制 / 文件操作 / 渗透测试 | 不规则 Grid 宽高跨越，毛玻璃卡片 |
| **笔记索引** | 19 篇带编号/图标卡 | hover 边框发光，点击跳转 list.html |
| **工具展示** | Burp Suite / Nmap / SQLMap | 渐变边框 + hover 上浮 |

### 3.2 学习笔记 (`list.html`)

**设计目标**：可动态更新的知识库，全状态覆盖

**技术实现**：

```
GitHub REST API
    │
    ├── 成功 → 解析文件名（编号/标题/图标）→ 动态构建目录
    │
    └── 失败/限流 → FILE_LIST_FALLBACK（19 篇硬编码回退列表）
                    │
                    ▼
              用户点击文件
                    │
                    ▼
        raw.githubusercontent.com 获取 .md 内容
                    │
                    ▼
            marked.js GFM 解析
                    │
                    ▼
            DOMPurify 净化（防 XSS）
                    │
                    ▼
            v-html 注入 + highlight.js 代码高亮
```

**状态覆盖**：

| 状态 | 展示 |
|------|------|
| **加载中** | 左侧骨架屏 (5 个脉冲占位) + 右侧骨架屏 (3 个文本块) |
| **空状态** | 图标 + "暂无笔记" 引导文案 |
| **错误** | 错误图标 + 错误描述 + "重试" 按钮 + "前往 GitHub" 链接 |
| **成功** | 动态目录 + Markdown 渲染内容 |

**交互细节**：
- **键盘快捷键**：`Ctrl + ←/→` 上一篇/下一篇，`ESC` 返回目录
- **URL Hash 持久化**：刷新页面保持阅读位置
- **移动端适配**：侧边栏折叠 + 悬浮切换按钮
- **GFM 完整支持**：表格、任务列表、代码块高亮、引用块

### 3.3 漏洞详情 (`detail.html`)

**设计目标**：最深入的内容页，体现专业知识储备

**6 大漏洞**：

| 漏洞 | 严重度 | OWASP 分类 | CWE 编号 |
|------|--------|------------|----------|
| SQL 注入 | 严重 | A03:2021 Injection | CWE-89 |
| XSS 跨站脚本 | 高危 | A03:2021 Injection | CWE-79 |
| CSRF 跨站请求伪造 | 高危 | A01:2021 Broken Access Control | CWE-352 |
| SSRF 服务端请求伪造 | 高危 | A10:2021 SSRF | CWE-918 |
| 文件上传漏洞 | 严重 | A04:2021 Insecure Design | CWE-434 |
| IDOR 不安全的直接对象引用 | 中危 | A01:2021 Broken Access Control | CWE-639 |

**每个漏洞的知识结构**：漏洞原理 → 攻击类型分类 → 终端风格攻击示例代码 → 防御方案清单

**交互**：`?id=sqli` URL 参数自动定位 + 桌面端 sticky 侧边栏导航

### 3.4 安全实验室 (`login.html`)

**设计目标**：展示前端表单工程能力

| 功能 | 实现方式 |
|------|----------|
| **登录/注册切换** | JavaScript 驱动面板显示/隐藏 + 激活态指示 |
| **邮箱验证** | 正则 `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` |
| **手机号验证** | 正则 `/^1[3-9]\d{9}$/`（中国大陆） |
| **密码强度** | 四级实时检测（弱/中/强/非常强），含 `role="progressbar"` + `aria-valuenow` |
| **密码一致性** | 实时比对确认密码字段 |
| **安全意识提示** | HTTPS 检查 / 钓鱼网站提醒 / 2FA 建议 / 密码管理器推荐 |

**安全承诺**：页面明确标注"仅作前端演示，不会向服务器发送凭据"

## 四、设计系统

### 4.1 视觉方向

选择 **暗黑赛博玻璃态 (Dark Cyber Glassmorphism)** 而非常见的浅色主题或 Material Design，原因是：
- 与「网络安全」主题气质契合
- 终端/代码编辑器风格映射，降低阅读疲劳
- 差异化设计，避免模板化外观

### 4.2 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| 最深底色 | `#020617` | 页面背景 |
| 卡片底色 | `#0f172a` | 毛玻璃卡片 |
| 主强调色 (Cyan) | `#06b6d4` | 链接、按钮、辉光 |
| 青色辉光 | `#22d3ee` | 强调装饰 |
| 紫色 (Purple) | `#a855f7` | 渐变、发光 |
| 红色 (Red) | `#ef4444` | 严重漏洞 / 错误状态 |
| 橙色 (Orange) | `#f97316` | 高危漏洞 / 警告 |
| 翠绿 (Green) | `#10b981` | 成功 / 安全 / 防御 |

### 4.3 核心特效

| 特效 | CSS 技术 | 性能考量 |
|------|----------|----------|
| **玻璃态卡片** | `background: rgba(15, 23, 42, 0.6)` + `backdrop-filter: blur(12px)` | GPU 合成层，不影响重排 |
| **霓虹辉光** | 多层 `box-shadow`（青/紫色） | 硬件加速，无重排 |
| **扫描线** | `body::after` + `repeating-linear-gradient` | `pointer-events: none` 不阻断交互 |
| **Glitch 动画** | `@keyframes` + `text-shadow` 偏移 + `clip-path` 切割 | hover 触发，不影响空闲帧率 |
| **SVG 噪点** | `feTurbulence` + `feColorMatrix` 内联 Data URL | 静态纹理，无运行时开销 |

### 4.4 响应式策略

| 断点 | 设备 | 布局变化 |
|------|------|----------|
| **≤ 768px** | 手机 | 单列布局，汉堡菜单，笔记侧边栏折叠 |
| **769–1024px** | 平板 | 2 列 Grid，笔记页显示侧边栏 |
| **≥ 1025px** | 桌面 | 详情页双栏 (280px + 1fr)，笔记页 (320px + 1fr) |

## 五、技术亮点与难点

### 5.1 动态内容 + 静态部署

**挑战**：纯静态站点如何展示「可动态更新」的学习笔记？

**方案**：客户端通过 GitHub REST API 实时抓取仓库文件列表与 Markdown 内容，marked.js 客户端渲染，DOMPurify 安全净化。API 限流时自动回退到硬编码的 19 篇预设列表，确保离线可用。

**成果**：仓库新增 .md 笔记后页面自动同步，无需重新部署。

### 5.2 XSS 防护闭环

**挑战**：`v-html` 注入用户不可控内容存在 XSS 风险

**方案**：三管齐下
1. **DOMPurify** 净化 marked.js 输出，剥离危险标签与事件处理器
2. **CSP 策略**：`script-src` 白名单限制、`frame-src 'none'`、`object-src 'none'`
3. **数据源可信**：仅从指定 GitHub 仓库的特定路径抓取

### 5.3 全状态覆盖

**挑战**：异步请求有四种状态，传统做法只处理成功

**方案**：每个异步视图覆盖 Loading / Empty / Error / Success 四种状态：
- 骨架屏加载动画（脉冲占位符）
- 空状态插画 + 引导文案
- 错误信息 + 重试按钮 + 备选链接
- 数据驱动的动态渲染

### 5.4 无构建工具的前端工程化

**挑战**：不使用 npm/webpack/vite 的前提下实现模块化

**方案**：
- CSS：单文件 1681 行，通过 `@layer` 和注释分隔组织
- JS：按职责拆分为 3 个独立文件（components / common / auth），通过 `<script>` 标签顺序加载
- Vue：单文件组件通过字符串模板实现，CDN 引入 Vue 3 运行时

## 七、不足与改进方向

| 不足 | 改进方向 |
|------|----------|
| 无真实后端，所有交互为前端模拟 | 可接入 Node.js/Python 后端实现真实鉴权与数据持久化 |
| CSS 单文件 1681 行过于庞大 | 可按组件/页面拆分成多个 CSS 文件并通过构建工具合并 |
| 无自动化测试 | 可添加 Playwright E2E 测试 + 视觉回归测试 |
| 依赖 CDN 可用性 | 可将关键依赖自托管，增加 SRI 完整性校验 |
| 无 PWA 离线支持 | 可添加 Service Worker + manifest 实现离线访问 |
| Vue 组件为字符串模板 | 可迁移至 Vue SFC + Vite 构建，获得更好的开发体验 |

## 八、学习收获

1. **从零到一构建完整前端项目**：从主题选定 → 设计系统 → 页面开发 → 响应式适配 → 部署上线的全流程
2. **深入理解 CSS 设计系统**：70+ 自定义属性、Bento Grid、复合动画特效，不再停留在"调颜色改边距"
3. **Vue 3 数据驱动思维**：将「GitHub API → 数据清洗 → 状态管理 → 模板渲染」流水线化
4. **安全意识内化**：CSP 策略、DOMPurify 净化、XSS 防御——这些不是"加分项"而是"基本素质"
5. **静态站点的工程化边界**：探索了无构建工具条件下，通过文件拆分、CDN 依赖、约定式路由实现工程化的上限

## 九、总结

**sec-notebook** 是一个综合运用 HTML5 + CSS3 + JavaScript + Vue 3 的 Web 前端课程作品。它以 Web 安全知识为内容载体，以暗黑赛博玻璃态为视觉方向，实现了 5 个页面、三级跳转、三端适配、动态内容渲染、完整表单验证等全部课程要求。
