# sec-notebook — Web 安全渗透测试学习平台

> **Web前端应用开发** 课程结课大作业 答辩材料

---

## 一、我做了一个什么项目

我做了一个叫 **sec-notebook** 的网站，主题是 Web 安全知识学习。

简单说就是：**一个介绍常见网站漏洞有哪些、怎么攻击、怎么防御的在线学习平台。**

打开网站能看到 5 个页面：首页、学习笔记、漏洞详情、安全实验室、关于项目。手机、平板、电脑都能正常看。

---

## 二、用到了哪些技术

这个项目没有用后端（没有 Python/Java/数据库那一套），是一个纯前端的静态网站。

| 技术 | 干什么用的 |
|------|-----------|
| HTML | 页面的骨架——标题、按钮、输入框这些元素的定义 |
| CSS | 页面的皮肤——颜色、布局、动画效果 |
| JavaScript | 页面的行为——点击按钮发生什么、表单怎么检查 |
| Vue 3 | 一个简化开发的框架——让数据和页面自动同步，不用手动操作 |
| Vite | 构建工具——把 Vue 的 .vue 文件编译成浏览器能跑的 JS |

换句话说：HTML 决定"有什么"，CSS 决定"长什么样"，JavaScript 决定"做什么"，Vue 让数据和界面自动联动，Vite 负责把 Vue 写的代码打包成最终产物。

项目使用 Vue 官方的 **单文件组件（.vue 文件）** 来组织代码——一个 .vue 文件里同时包含模板（HTML）、逻辑（JS）和样式（CSS），结构清晰。开发时用 `npm run dev` 启动本地服务器（支持热更新，改了代码立刻能看到效果），发布时用 `npm run build` 打包到 dist 目录。

## 三、5 个页面分别是什么

### 3.1 首页

打开网站看到的第一个页面。上面有项目的标题（带了一个故障抖动的动画效果），中间用卡片展示了 5 个安全知识模块，下面列出了全部 19 篇学习笔记的入口。配色是暗黑色系 + 青色发光，和"黑客/安全"的主题比较搭。

### 3.2 学习笔记（最核心的页面）

这个页面会自动从我的 GitHub 仓库（一个存笔记的网盘）把 19 篇 Markdown 笔记抓下来，显示在左边的目录里。点左边任意一篇，右边就会把 Markdown 内容渲染成漂亮的网页格式——有代码高亮、表格、标题层级。

这里做了几个细节处理：
- **加载中**：显示灰色骨架屏占位（灰色条在闪烁），告诉用户在加载
- **加载失败**：显示重试按钮，不会白屏卡死
- **API 挂了**：GitHub 有时候会限流不让抓，代码会自动切换到内置的备用列表

### 3.3 漏洞详情

深度讲解了 6 种常见的 Web 漏洞：
1. **SQL 注入** — 在输入框里塞 SQL 语句去偷数据库
2. **XSS 跨站脚本** — 在网页里注入恶意 JavaScript 代码
3. **CSRF** — 伪造用户请求，让用户在不知情的情况下执行操作
4. **SSRF** — 利用服务器去访问内网的机器
5. **文件上传漏洞** — 上传恶意文件到服务器
6. **IDOR** — 通过改 URL 里的 ID 去访问别人的数据

每个漏洞都按同样的结构来讲：**是什么原理 → 怎么攻击（有代码示例）→ 怎么防御**。

### 3.4 安全实验室

一个登录和注册的表单页面。虽然不会真的把数据发给服务器，但前端该做的验证都做了：
- 邮箱格式对不对
- 手机号是不是 11 位
- 密码够不够强（弱/中/强/非常强四个等级实时显示）
- 两次密码输得一不一样

## 四、页面长什么样（设计）

整体风格选了**暗黑色系**，类似代码编辑器或终端的配色，跟"安全/黑客"主题协调。不是那种白底蓝按钮的常见模板。

主要做了几个视觉效果：
- 卡片是半透明的毛玻璃质感
- 重要文字有青色或紫色的发光效果
- 整页有一层很淡的扫描线叠加（不仔细看注意不到）
- 标题鼠标移上去会抖动错位一下

布局上用了一个叫 Grid 的技术，让卡片能自动适应屏幕宽度——手机上单列、平板上两列、电脑上三列或更多。

## 五、遇到的困难和怎么解决的

### 困难 1：网站是静态的，但笔记要能更新

正常的做法是搞个后端服务器 + 数据库。但课程要求是纯前端项目，我就换了个思路：让网页启动时自动去 GitHub 抓我的笔记文件。GitHub 免费提供了一个接口（API），可以读取仓库里的文件列表和内容。这样我在 GitHub 上更新了笔记，网站自动就同步了，不需要重新部署。

如果 GitHub 接口挂了（免费接口有次数限制），代码会自动切到一份备份列表，保证页面还能用。

### 困难 2：渲染 Markdown 时的安全问题

Markdown 转成 HTML 后，如果直接插到页面上，可能会有恶意脚本被执行（XSS 攻击）。我的处理方式是：在插入页面前先用一个叫 DOMPurify 的工具把 HTML "洗一遍"，过滤掉危险的标签和属性。同时在页面里配置了 CSP（内容安全策略），限制了哪些来源的脚本可以执行。

### 困难 3：快速点笔记时的乱序问题

用户快速点左边目录切换笔记时，如果前一个请求还没返回、后一个已经返回了，页面会先显示新的、再被旧的结果覆盖，出现内容闪烁。

解决方式：每次发新请求前，先取消上一个还没完成的请求（用了一个叫 AbortController 的机制）。这样旧请求的结果回来时会被直接丢弃，不会干扰当前显示。

## 附：相关技术原理

### Vue 3 响应式系统

Vue 3 通过 Proxy 代理数据对象，在数据被读取时收集依赖（哪些地方用了这个数据），在数据被修改时触发更新。类似于指针的概念——多个地方引用同一块内存，改了就都反映出来，只不过 Vue 把这个过程自动化了，开发者不需要手动操作 DOM。

### v-html 与 XSS 风险

`v-html` 将字符串作为 HTML 直接插入页面，浏览器会解析执行其中的标签和脚本。如果字符串包含 `<script>` 或 `onclick` 等事件属性，就会执行恶意代码。防护手段是在插入前用 DOMPurify 净化——只保留安全标签（`<p>`、`<h1>`、`<table>` 等），剥离脚本和事件处理器。

### GitHub API 数据获取流程

JavaScript 的 `fetch()` 函数向 GitHub REST API 发起 HTTP 请求，GitHub 返回仓库文件列表的 JSON 数据。解析 JSON 获得文件名数组后渲染到目录中。API 请求失败时（如触发频率限制），catch 捕获错误，回退到内置的 19 篇文件列表。

### 响应式布局原理

CSS 媒体查询（Media Query）根据屏幕宽度条件应用不同样式规则。配合 CSS Grid 和 Flexbox 布局，元素能根据可用空间自动调整排列方式——大屏幕上多列并排，小屏幕上堆叠为单列。

### AbortController 机制

`fetch()` 接受 `signal` 参数，调用 `abort()` 可取消未完成的网络请求。取消后 Promise 进入 reject 状态，错误类型为 AbortError，在 catch 中识别并静默处理。用于解决快速切换内容时的请求竞态问题。

### Markdown 渲染流程

marked.js 将 Markdown 文本解析为 HTML 字符串（如 `**粗体**` → `<strong>粗体</strong>`），然后通过 `v-html` 插入页面。highlight.js 遍历页面中的代码块，根据语言标识加上语法高亮。两个库协作完成从 .md 文件到可视化页面的转换。

### Vue 单文件组件（.vue 文件）

一个 `.vue` 文件包含三部分：`<template>` 放 HTML 模板、`<script setup>` 放 JavaScript 逻辑、`<style>` 放 CSS 样式。Vite 在构建时把 .vue 文件编译成浏览器可执行的 JS。对比早期直接用字符串拼模板的做法，.vue 文件有语法高亮、编译检查、组件作用域隔离等优势。

### CSP 安全策略

Content-Security-Policy 在 HTML 的 `<meta>` 标签中声明脚本/样式/图片来源的白名单。浏览器执行前检查——白名单内的来源放行，其他来源（包括 XSS 注入的恶意脚本）拒绝执行。这是在浏览器层面的防御，即使 HTML 净化漏掉了某些恶意内容，CSP 仍能拦截。
