## 一、项目概述

### 选题背景

随着 Web 应用复杂度提升，安全漏洞威胁日益严峻。本项目以 OWASP Top 10 为框架，构建了一个系统化的 Web 安全学习平台，目标是以攻促防——通过理解攻击原理来掌握防御方法。

### 项目定位

**sec-notebook** 是一个纯前端静态网站，无需后端服务器或构建工具，可直接部署至 Cloudflare Pages。

| 指标 | 数值 |
|------|------|
| 页面数量 | 5 个 |
| 页面层级 | 三级（首页 → 笔记库 → 详情） |
| 学习笔记 | 19 篇（支持动态从 GitHub 仓库同步） |
| 漏洞分析 | 6 大 OWASP 漏洞 |
| 响应式适配 | 手机 / 平板 / 桌面 三端 |
| 版本控制 | Git 管理，14 次迭代提交 |

## 二、技术架构

### 技术选型

| 技术 | 用途 |
|------|------|
| HTML5 | 语义化结构：Header / Nav / Main / Footer 统一布局 |
| CSS3 | 70+ 自定义属性、Grid + Flexbox 响应式布局、动画系统 |
| JavaScript (ES6+) | 导航交互、表单验证、Toast 通知、IntersectionObserver 滚动动画 |
| Vue 3 (CDN) | 学习笔记数据驱动渲染、搜索筛选 |

## 三、页面功能展示

### 3.1 首页

- Hero 区域：Glitch 故障动画标题、霓虹辉光徽章
- 核心数据统计：笔记数量 / OWASP 分类 / 严重度分级
- Bento Grid 布局：5 个核心安全模块（注入攻击 / XSS / 访问控制 / 文件操作 / 渗透测试）
- 19 篇笔记索引卡片 + 3 个常用渗透工具卡片

### 3.2 学习笔记（核心功能页）

- **动态数据源**：通过 GitHub REST API 从仓库 `wqnmlgb151/study` 实时抓取 Markdown 笔记列表
- **双栏布局**：左侧目录导航 + 右侧 Markdown 渲染区
- **GFM 完整支持**：表格、代码高亮、任务列表、引用块
- **全状态覆盖**：骨架屏加载态 / 空状态引导 / 错误重试 / 成功渲染
- **容错机制**：API 限流时自动回退到内置的 19 篇硬编码列表
- **键盘快捷键**：`Ctrl + ←/→` 翻页、`ESC` 返回目录
- **URL Hash 持久化**：刷新页面保持阅读位置

### 3.3 漏洞详情

覆盖 6 大 OWASP 漏洞，每个漏洞按"原理 → 攻击类型 → 防御方案"结构组织：

| 漏洞 | 严重度 | CWE 编号 |
|------|--------|----------|
| SQL 注入 | 严重 | CWE-89 |
| XSS 跨站脚本 | 高危 | CWE-79 |
| CSRF 跨站请求伪造 | 高危 | CWE-352 |
| SSRF 服务端请求伪造 | 高危 | CWE-918 |
| 文件上传漏洞 | 严重 | CWE-434 |
| IDOR 不安全的直接对象引用 | 中危 | CWE-639 |

支持 `?id=` URL 参数定位 + 侧边栏内容导航。

### 3.4 安全实验室

- 登录 / 注册标签切换，JavaScript 驱动
- 完整表单验证：邮箱格式、手机号（中国大陆）、密码强度四级检测（含 ARIA `role="progressbar"`）、密码一致性
- 密码强度检测：综合评估长度、大小写、数字、特殊字符
- 安全意识提示面板
- 页面明确标注"仅作前端演示"，不涉及真实鉴权

- 项目背景与"以攻促防"理念

## 四、设计系统

### 视觉方向

采用"暗黑赛博玻璃态"（Dark Cyber Glassmorphism），理由：
- 与网络安全主题气质契合
- 终端/代码编辑器风格，降低长时间阅读疲劳
- 区别于常见的浅色 Material Design 模板

### 核心特效

| 特效 | 实现方式 |
|------|----------|
| 毛玻璃卡片 | `backdrop-filter: blur()` + 半透明背景 |
| 霓虹辉光 | 多层 `box-shadow`（青色 + 紫色） |
| 扫描线叠加 | `body::after` + `repeating-linear-gradient`，`pointer-events: none` 不阻断交互 |
| Glitch 动画 | `@keyframes` hover 触发文字错位 + 色彩分离 |
| SVG 噪点纹理 | `feTurbulence` + `feColorMatrix` 内联 Data URL |

### 响应式断点

| 断点 | 设备 | 布局变化 |
|------|------|----------|
| ≤ 768px | 手机 | 单列、汉堡菜单、侧边栏折叠 |
| 769–1024px | 平板 | 两列 Grid |
| ≥ 1025px | 桌面 | 完整双栏布局 |

## 五、技术难点与解决方案

### 5.1 静态站点的动态内容更新

**问题**：纯静态站点如何展示可持续更新的学习笔记？

**方案**：
- 客户端通过 GitHub REST API 实时获取仓库文件列表
- marked.js 客户端渲染 Markdown
- API 不可用时自动回退到内置列表，保证离线可用

**效果**：仓库新增笔记后页面自动同步，无需重新部署。

### 5.2 XSS 防护

**问题**：使用 `v-html` 渲染 Markdown 存在 XSS 风险。

**方案（三层防护）**：
1. DOMPurify 净化 marked.js 输出，剥离危险标签和事件处理器
2. 每页配置 Content-Security-Policy meta 标签，限制脚本来源
3. 数据源仅限指定 GitHub 仓库的特定路径

### 5.3 异步请求竞态处理

**问题**：快速切换笔记时，先发出的请求可能后返回，覆盖最新内容。

**方案**：使用 AbortController 在新请求发起前取消上一个未完成的 fetch，并在 catch 中识别 `AbortError` 静默处理。

### 5.4 全状态覆盖

每个异步视图覆盖 Loading / Empty / Error / Success 四种状态，而非仅处理成功路径。

## 七、自我评价与改进方向

### 做得好的

- 从零到一独立完成完整前端项目，涵盖设计 → 开发 → 部署全流程
- CSS 设计系统有完整的自定义属性体系和响应式策略
- XSS 防护（DOMPurify + CSP）和全状态覆盖体现工程意识
- 纯静态 + CDN 方案实现零依赖安装

### 可以改进的

| 不足 | 改进方向 |
|------|----------|
| 无真实后端 | 可接入后端实现真实鉴权与数据持久化 |
| CSS 单文件过大（1681 行） | 按组件拆分并通过构建工具合并 |
| 无自动化测试 | 可添加 Playwright E2E 测试 |
| 依赖外部 CDN | 关键依赖可自托管并添加 SRI 校验 |
| 无 PWA 离线支持 | 添加 Service Worker 实现离线访问 |

## 八、总结

sec-notebook 综合运用 HTML5 + CSS3 + JavaScript + Vue 3 技术栈，以 Web 安全知识为内容载体，实现了 5 个页面、三级页面跳转、三端响应式适配、动态内容渲染、完整表单验证等全部课程要求。

项目在 GitHub 开源，支持一键部署至 Cloudflare Pages，全程使用 Git 版本控制管理 14 次迭代。
