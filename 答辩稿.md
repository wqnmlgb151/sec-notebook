# sec-notebook 答辩材料

> Web 安全渗透测试学习平台 | Vue 3 + Vite + Vue Router 4 + GitHub API

---

## 一、整体介绍

**sec-notebook** 是一个面向 Web 安全初学者的在线学习平台，系统化讲解常见 Web 漏洞的原理与防御方案。采用 **Vue 3 SPA（Vite + Vue Router 4）** 架构，纯前端部署于 Cloudflare Pages。

一句话：**用 Vue 3 构建的单页应用，5 个页面通过客户端路由切换，数据从 GitHub 动态抓取的 Web 安全知识库。**

## 三、功能演示（按页面模块）

### 3.1 首页（`/`）

- **Hero 首屏**：Glitch 故障动画标题 + 霓虹辉光徽章 + 背景光球
- **数据统计**：笔记数量 / 专项专题 / GFM 渲染 / GitHub 数据源
- **Bento Grid**：5 个核心安全模块卡片（注入攻击 / XSS / 访问控制 / 文件操作 / 渗透测试方法论）
- **笔记索引**：19 篇笔记入口卡片，点击跳转 `/notes#文件名`
- **工具展示**：Burp Suite / Nmap / SQLMap 三卡片

### 3.2 学习笔记（核心功能页，`/notes`）

- **动态数据源**：页面启动时通过 GitHub REST API 抓取仓库文件列表
- **容错设计**：API 限流时自动回退到内置 19 篇文件列表
- **双栏布局**：左侧目录导航 + 右侧 Markdown 渲染
- **Markdown 渲染管道**：fetch .md → marked.js 解析 → DOMPurify 净化 → v-html 渲染 → highlight.js 代码高亮
- **全状态覆盖**：骨架屏加载 → 空状态引导 → 错误重试 → 成功渲染
- **键盘导航**：Ctrl+←/→ 翻页、ESC 返回目录
- **Hash 持久化**：刷新保持阅读位置，直接分享链接给他人
- **并发预加载**：3 路并发 + 60s 超时策略，后台静默缓存全部笔记

### 3.3 漏洞详情（`/detail`）

6 大 OWASP 漏洞深度分析，按"原理 → 攻击类型 → 防御方案"结构组织：

| 漏洞 | CWE | 严重度 |
|------|-----|--------|
| SQL 注入 | CWE-89 | 严重 |
| XSS 跨站脚本 | CWE-79 | 高危 |
| CSRF 跨站请求伪造 | CWE-352 | 高危 |
| SSRF 服务端请求伪造 | CWE-918 | 高危 |
| 文件上传漏洞 | CWE-434 | 严重 |
| IDOR 不安全的直接对象引用 | CWE-639 | 中危 |

- **数据驱动渲染**：6 个漏洞从 JS 数组中 v-for 生成，新增漏洞只需加一个对象
- **侧边栏滚动高亮**：rAF 节流 scroll 事件 + getBoundingClientRect 实时检测阅读位置
- **URL 参数定位**：`/detail?id=xss` 自动滚动到指定漏洞
- **终端模拟**：攻击示例以终端风格展示（SQL 注入、XSS、SSRF 含具体攻击命令）

### 3.4 安全实验室（`/lab`）

- 登录/注册标签切换（v-show，无 DOM 重建）
- 表单验证：邮箱格式、手机号（正则）、密码强度四级检测
- 密码强度算法：综合评估长度 + 大小写 + 数字 + 特殊字符
- Toast 消息系统：成功/错误/信息提示，自动消失

### 3.5 关于项目（`/about`）

背景、目标、技术栈表格、设计理念标签云（10 个徽章）、四阶段开发时间线。

## 四、路由设计

本项目采用 **Vue Router 4 客户端路由**，5 个页面通过 URL 路径切换，无需刷新。

### 路由表

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | HomePage.vue | 首页 |
| `/notes` | NotesPage.vue | 学习笔记（含 hash 定位笔记） |
| `/detail` | DetailPage.vue | 漏洞详情（含 ?id= 参数定位） |
| `/lab` | LabPage.vue | 安全实验室 |
| `/about` | AboutPage.vue | 关于项目 |
| `/html/:page.html` | redirect | 旧 URL 兼容（自动跳转到新路径） |
| `/*` | redirect → `/` | 404 兜底 |

### 关键设计点

| 机制 | 实现 |
|------|------|
| **客户端路由** | `createWebHistory()` — 无 `#` 的干净 URL |
| **懒加载** | `() => import(...)` — 每个页面独立 chunk，首屏仅加载首页 |
| **导航高亮** | `<router-link>` 自动添加 `.router-link-active` 类 |
| **Hash 定位** | 首页笔记卡片链接 → `/notes#文件名`，NotesPage 通过 `route.hash` 解析 |
| **参数定位** | detail 页读取 `route.query.id` 自动滚动到对应漏洞 |
| **旧 URL 兼容** | `/html/list.html` → 自动 301 跳转 `/notes` |
| **滚动行为** | `scrollBehavior` 控制页面切换后滚动到顶部或锚点 |

## 五、核心组件讲解

### 5.1 NotesPage.vue — 学习笔记核心（~280 行）

**职责**：管理整个笔记系统的状态和渲染。

**核心数据**（全部用 `ref()` 声明）：
- `files` — 笔记文件列表（初始为内置 19 篇，API 成功后更新）
- `sel` — 当前选中的笔记
- `renderedHTML` — Markdown 渲染后的 HTML（经 DOMPurify 净化）
- `isLoading` / `isDirLoading` / `err` — 加载/错误状态
- `showSidebar` — 移动端侧边栏开关

**核心方法**：
- `loadMarkdown(file)` — fetch → marked.parse → DOMPurify.sanitize → v-html 渲染 → hljs.highlightAll
- `selectFile(f)` — 选择笔记并更新 URL hash
- `goToPrev()` / `goToNext()` — 上一篇/下一篇翻页
- `preloadNotes()` — 3 路并发预加载 + 60s 超时

### 5.2 AppHeader.vue — 全局导航栏

- `<router-link>` 自动获取当前路由并高亮对应导航项
- 汉堡菜单状态由 Vue `ref` 驱动（切换 body overflow 防止背景滚动）
- 路由切换时自动关闭移动端菜单
- rAF 节流滚动监听，动态添加 `header-scrolled` 样式

### 5.3 DetailPage.vue — 漏洞详情数据驱动

- 6 个漏洞从 `articles` 数组中 v-for 渲染，含 terminalLines 子数组支持终端模拟
- 侧边栏滚动高亮：`getBoundingClientRect` + rAF 节流，底部边界检测
- `route.query.id` 自动定位

### 5.4 LabPage.vue — 表单验证与密码强度

- `ref` + `reactive` 管理表单数据和错误状态
- `computed` 实时计算密码强度（四级：弱/中/强/非常强）
- 登录/注册面板用 `v-show` 切换，保留表单输入状态
- 使用 `useToast()` composable 显示验证结果

### 5.5 Router 懒加载机制

```js
const routes = [
  { path: '/', component: () => import('../views/HomePage.vue') },
  { path: '/notes', component: () => import('../views/NotesPage.vue') },
  // ... 每个页面独立 chunk
]
```

Vite/Rollup 构建时每个 `import()` 拆分为独立 JS 文件，首屏仅加载 HomePage（~11KB）和公共 vendor（~98KB），NotesPage 的大依赖（marked + highlight.js，~131KB）在用户点击"学习笔记"时才按需加载。

## 六、Vue 3 核心 API

### ref() vs reactive() — 本项目选择 ref()

| 特性 | ref() | reactive() |
|------|-------|------------|
| 基本类型 | ✅ 支持 | ❌ 不支持 |
| 整体替换 | ✅ `files.value = newArr` | ❌ 丢失响应性 |
| 解构 | ✅ 需 `.value` | ❌ 丢失响应性 |
| 模板解包 | ✅ 自动 | ✅ 自动 |

本项目需要在 API 返回后整体替换数组（`files.value = newArray`），因此全部使用 `ref()`。

### 实际使用

```js
const files = ref([...filesArr])     // 可整体替换
const sel = ref(null)                // 可整体替换
const isLoading = ref(false)
const showSidebar = ref(true)

// computed 派生状态 — 不用手动维护
const currentIndex = computed(() => files.value.findIndex(...))
const hasPrev = computed(() => currentIndex.value > 0)

// watch 替代轮询 — 声明式响应
const stopWatch = watch(files, (val) => {
  if (val.length > 0) { stopWatch(); /* 定位目标笔记 */ }
})
```

### 技术架构演进

| 版本 | 架构 | 问题 |
|------|------|------|
| v1 CDN 方式 | 5 个独立 HTML，Vue 3 通过 CDN 加载 | 无构建工具，手动管理依赖 |

2. **无自动化测试**：所有功能靠浏览器手动验证
3. **highlight.js 主题通过 CSS 全局引入**：未按需加载语言包主题

### 改进方向

1. 引入后端（Node.js/Python）实现真实用户系统
2. 添加 Playwright E2E 测试覆盖核心流程
3. 添加 Service Worker 实现 PWA 离线访问
